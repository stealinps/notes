#Big O notation
-Measures the efficiency of an algorithm when the input grows commonly annotate by "n"
-O(1),O(log n),O(n),O(nlogn),O(n²),O(2^n),O(n!)

#Time complexity
-Measured in unit of steps/How many steps it took to complete the algorithm
-O(1) means instantly
-O(n) means it grows linearly 
-O(log n) means it grows slower than O(n) [aka logarithmically]
-O(n²) means it grows quadratically
-O(nlogn) means it grows faster than O(log n) but slower than O(n²)
-O(2^n) means it grows exponentially faster than O(n²)
-O(n!) grows faster than O(2^n) at big n value

##Object Oriented Programming(OOP)
-OOP is a programming paradigm which organizes code into segment of objects representing entity or concept

#Terminologies
Functions:Modular segment of code that perform a task
Structures:Grouping different data types under a single name enabling creating of user-defined data types
Object:Contains attributes(state) and functions(behaviour) bundled together that allows manipulation of data. Objects can be any physical and logical object
Class:A blueprint/template to create objects/defining the attributes and funstions of an object
Abstraction:Providing necessary information only and leaving behind the too specific details
Encapsulation:Bundling data and methods into single class
Inheritance:Extending the attributes of existing classes to new class
Polymorphism:Ability to use a function in different ways based on input and data type

OOP offers code organization, modularity and reusability
Able to mimics real world cases more effectively

factorial
#include <iostream>
int factorial(int n){
	if(n==0||n==1){
		return 1;
	}
	else{
		return n*factorial(n-1);
	}
}

int main(){
	int num=5;
	std::cout<<"Factorial of "<<num<<" is: "<<factorial(num)<<std::endl;
	return 0;
}

Data Types(DT) VS Abstract Data Types(ADT)

DT:
-Defines a certain domain of values
-Defines Operations allowed on those values

e.g:
int num
-takes only int values
-allowed operations includes:addition, subtraction, multiplication, modulus and more

float num
-takes only float values
-allowd operations includes:addition, subtraction, multiplication and more but doesnt includes modulus

User Defined Data Types(UDT)
-primitive data types:int, bool, char, byte, short, long, float, double
(data types included with the programming language)
-UDT is the data types that the user defined themselves
-like struc,enum and union
e.g:
struct point{
	int x;
	int y;
};
Abstract Data Types(ADT)
-ADT is a model used to represent data structures by its behaviour, operations and posibble values likes UDT
-by using functions
-it is like a psuedocode of a data structure
-Key characteristics of ADT includes:
	-Encapsulation
	-Abstraction
	-Modularity
e.g:
stack ADT
-A stack consists of elements of same type arranged in a sequential order
Operations:
initialize()-initialize
push()-insert an element
pop()-delete an element
isEmpty()-checks if the stack is empty
isFull()-checks if its full
