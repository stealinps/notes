#Big O notation
-Measures the efficiency of an algorithm when the input grows commonly annotate by "n"
-O(1),O(log n),O(n),O(nlogn),O(n²),O(2^n),O(n!)

#Time complexity
-Measured in unit of steps/How many steps it took to complete the algorithm
-O(1) means instantly
-O(n) means it grows linearly 
-O(log n) means it grows slower than O(n) [aka logarithmically]
-O(n²) means it grows quadratically
-O(nlogn) means it grows faster than O(log n) but slower than O(n²)
-O(2^n) means it grows exponentially faster than O(n²)
-O(n!) grows faster than O(2^n) at big n value

##Object Oriented Programming(OOP)
-OOP is a programming paradigm which organizes code into segment of objects representing entity or concept
#Terminologies
Functions:Modular segment of code that perform a task
Structures:Grouping different data types under a single name enabling creating of user-defined data types
Object:Contains attributes(state) and functions(behaviour) bundled together that allows manipulation of data. Objects can be any physical and logical object
Class:A blueprint/template to create objects/defining the attributes and funstions of an object
Abstraction:Providing necessary information only and leaving behind the too specific details
Encapsulation:Bundling data and methods into single class
Inheritance:Extending the attributes of existing classes to new class
Polymorphism:Ability to use a function in different ways based on input and data type
Public:Accessible Everywhere Protected:Accessible by derived subclass Private:Only accessible inside its own class

OOP offers code organization, modularity and reusability
Able to mimics real world cases more effectively

Stack(LIF0)

-Fundamental data structure with LIFO principle
-The Last data In will be the Firdt to be Removed
-Think like a pile of clothes, where you need to get to the first item, you have to remove the clothes one by one from up to down
-Stack as the name suggest follows the same principle
-Accessing an element at n require O(n) time unlike array with O(1)
-Operations of Stack:
	-Push(element) //push an element to the top
	-Pop() //remove the top element
	-Peek()/Top() //view the top element without removing it
	-Clear() //Clear the stack
	-Display() //View all the elements in the stack

example:	
Stack Test(10); ______
Test.Push(1);	|  3 |

Test.Push(3);	|  1 |
		------

Test.Pop();     ------

		------
Test.Push(4);	______
		|  4 |
		|  1 |
		------

{
	if(CurrentPosition<MaxCapacity-1){
		CurrentPosition=CurrentPosition+1;
		Array[CurrentPosition]=value;
	}
	else

}

{
	int Result;
	if(CurrentPosition>=0){
		Result=Array[CurrentPosition];
		CurrentPosition--;
		return Result;
	}
	else

		std::cout<<"Stack Underflow\n";
		return -1;
	}
}

{
	int Result;

		return Result;

}

-Fundamental data structure with FIFO principle
-Insert at rear, remove at front
-The Last data In will be the last to be Removed
-Accessing an element at n require O(n) time 
-Operations of Queue:
	Front:Remove()
	Rear:Insert()
	Count:Keeps track of how many elements in the queue
example:

Insert(7)  stack overflow
remove()   [99,8,2,10,] count=4 , all data moves to fill the gap

Circular Queue, the front and rear moves
-First Index and Last Index is connected

class Queue{

		int Rear;
		int Count;
		int ArraySize;
	};
Queue::Queue(){
	ArraySize=6;
	Array=new int[ArraySize];//access the *Array in private
	Front=0;
	Rear=ArraySize-1;
	Count=0;
	std::cout<<"Queue created \n";
}
Queue::Queue(int QueueSize){
	Array=new int[QueueSize];
	ArraySize=QueueSize;
	Front=0;
	Rear=ArraySize-1;
	Count=0;
	std::cout<<"Queue 2 created \n";
}
Queue::~Queue(){
	delete []Array;
	std::cout<<"Queue bombed to death";
}
void Queue::Insert(int Val){
	if(Count==ArraySize){
		std::cout<<"Queue is FULL!\n";
		return;
	}
	if(Rear==ArraySize-1)
		Rear=0;
	else
		Rear++;
	Array[Rear]=Val;
	Count++;
}
int Queue::Remove(){
	int Temp;
	if(Count==0){
	std::cout<<"Queue is empty!\n";
		return -1;
	}
	Temp=Array[Front];
	if(Front==ArraySize-1) Front=0;
	else Front++;
	Count--;
	return Temp;
}
int Queue::First(){
	if(Count==0){
		std::cout<<"Queue is empty!\n";
		return -1;
	}
	return Array[Front];
}
void Queue::Clear(){
	Front=0;
	Rear=ArraySize-1;
	Count=0;
}
void Queue::Display(){
    if(Count==0){ std::cout<<"Queue is empty!\n"; return; }
    for(int i=0;i<Count;i++)
        std::cout << Array[(Front+i) % ArraySize] << std::endl;
}
